ActionEngine: The Death of Step-by-Step AI Browsing

Microsoft Research + Georgia Tech High Relevance

ActionEngine fundamentally reimagines how AI agents interact with web applications. Instead of the expensive screenshot→LLM→action loop (10+ LLM calls per task), it builds a state machine memory map offline, then generates a complete Python program to execute the entire task with ~1.8 LLM calls.

How It Works

Two-agent architecture (training-free):

  • Crawling Agent (offline): Systematically explores the GUI, builds a directed graph where nodes = pages/states and edges = executable operations. Uses "dynamic atoms" to keep the graph compact (~20-30 states per app).
  • Execution Agent (online): Takes task + state graph, generates a complete Python program sketch with UI_CALL placeholders, statically links them to graph paths, compiles to a typed execution plan, runs deterministically.
"Logic is encoded in Python (deterministic), not in vision reasoning (expensive + unreliable). The LLM is used for high-level planning, not repetitive step-by-step decisions."
Ali Implications
  • Architecture pattern: State machine memory is directly applicable to Ali's conversation engine — negotiation flows can be modeled as state graphs with deterministic transitions, reducing LLM calls per coaching session
  • Cost structure: Moving from reactive per-turn LLM calls to planned program execution could dramatically reduce token costs at scale
  • Contract parser: When parsing contract documents, pre-building a structural map before analysis (like ActionEngine's crawling phase) would improve accuracy and reduce costs

Agent Memory: From Conversation Logs to Execution Traces

DAIR.AI + Context+ + Slack Research Critical

The field is converging on a fundamental insight: evaluating agent memory on chatbot-style dialogues is wrong. Real agents interact with databases, code executors, and web interfaces, generating machine-readable trajectories — not conversational text.

The Paradigm Shift

Old ModelNew Model

Summarize text for recallCompress trajectories into semantic insights

Memory = what was saidMemory = what was done + outcomes

Context+ Tool: Solving Context Bloat

Open-source MCP server that transforms codebases into searchable, hierarchical feature graphs using tree-sitter AST parsing and spectral clustering. Instead of dumping raw files, it delivers targeted structural and semantic information. In testing: 6.5K fewer tokens and half the resolution time.

Snapchat's Auton Framework: Reflector-Driven Memory

Addresses LLM statelessness through a Reflector-Driven Consolidation Protocol inspired by biological memory consolidation. Raw event streams are compressed into semantic insights that persist across sessions without model retraining. Execution optimization via Cognitive Map-Reduce parallelizes independent reasoning steps.

Economic Theory

The Verification Economy: Human Bandwidth as the Binding Constraint

Import AI 447 + NBER Working Papers Opportunity

"Some Simple Economics of AGI" (MIT, WashU, UCLA)

The AGI transition is modeled as the collision of two racing cost curves: an exponentially decaying Cost to Automate and a biologically bottlenecked Cost to Verify. The binding constraint on growth is not intelligence — it's human verification bandwidth.

"We are moving from an era where our worth was defined by our capacity to build and discover, to an era where our survival depends on our capacity to steer, understand, and stand behind the meaning of what is created."

The Hollow Economy risk: Agents consume real resources to produce output that satisfies measurable proxies while violating unmeasured intent. This creates "counterfeit utility" — high nominal output but collapsing realized value.

Acemoglu/Autor: Pro-Worker AI (NBER W34854)

Distinguishes five categories of AI change. Only "new task-creating" AI is unambiguously pro-worker. The paper identifies market failures in current AI development (automation bias in VC/tech) and proposes 9 policy directions favoring collaborative AI.
