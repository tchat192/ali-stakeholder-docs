Concept

What Are Design Tokens?

Design tokens are the single source of truth for visual design decisions. Instead of hardcoding #8A41E6 everywhere, we define it once as --ali-accent and reference that everywhere. Change it in one place, it updates everywhere.

"One change in Figma = One change in code = Updates everywhere"

Implementation

How It Looks in Code

/* ali-product-brief.css */
:root {
  /* Extracted from Figma */
  --ali-foreground: #121212;     /* black/black */
  --ali-card: #FFFFFF;          /* white/white */
  --ali-secondary: #7C7C7C;     /* grey/500 */
  --ali-muted: #5E5E5E;         /* grey/600 */
  --ali-border: #DBDBDB;        /* grey/300 */
  --ali-card-subtle: #FBFBFB;  /* grey/50 */
  --ali-background: #F5F3F0;   /* page bg */
}

.button-primary {
  background: var(--ali-foreground);
  color: var(--ali-card);
  box-shadow: var(--ali-shadow-tab);
}
Guidelines

Token Best Practices

Do

  • Use Figma Variables for all colors
  • Name tokens semantically (grey/500, not "placeholder-text")
  • Use consistent spacing scale (4, 8, 12, 16...)
  • Define all typography as tokens
  • Include effects (shadows) as tokens

Don't

  • Hardcode hex colors directly in frames
  • Use random spacing values (7px, 13px)
  • Create one-off tokens for single uses
  • Mix naming conventions (grey-500 vs grey/500)
  • Forget to update token when design changes
