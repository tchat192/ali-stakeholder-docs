Reference

States to Design Per Screen

For each screen, consider whether these states need unique designs:

Interactive States

  • Default (initial load)
  • Hover (cursor over)
  • Active/Pressed
  • Disabled (grayed out)
  • Focus (keyboard nav)

Data States

  • Loading (skeleton/spinner)
  • Empty (no data yet)
  • Error (something failed)
  • Success (action completed)
  • Partial (incomplete data)
