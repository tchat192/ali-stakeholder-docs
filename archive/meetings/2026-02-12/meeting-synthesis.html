New Coaching Moves Identified
Emerged from the meeting — not in pre-meeting choreography or existing catalog

1. Transparency Calibration by Negotiation Type

"It's not honesty, it's transparency."— Tom Hilgeman (verbatim, Plaud transcript)
"Bargaining should be like a two three. Trading should be like a four or five. Creating and partnering is more like a seven, eight, because you're never going to be a 10 in these situations."— Mark Mirra (verbatim, Plaud transcript)

Ali should modulate how much information it coaches users to share based on the negotiation type. Specific scale: Bargaining = 2-3/10, Trading = 4-5/10, Creating/Partnering = 7-8/10, never 10. This replaces the flat "honesty" parameter with a dynamic, type-driven transparency model. Mark also coined "leverage tell" for when Ali reveals position-weakening information (e.g., "these are from a canceled order").

Catalog status: NEW. Not in FRAMEWORK_CATALOG. Should be added as a coaching move.

2. Term Prioritization Question

"Ali needed to ask one more question earlier in the process: which terms the user needs to maximize and where they can be flexible."— Mark Mirra

A required early-conversation question that prevents Ali from suboptimizing — directly caused by the backwards pricing tiers in Brake Rotors. This is the fix Mark articulated.

Catalog status: NEW. Extension of A2 (Zones) — explicit prioritization before strategy generation.

3. Four Required Diagnostic Questions

"How many terms are being negotiated and what they are. How many alternatives are in the marketplace, are they attractive? What's the intent in this contract, one-off to long-term? And what's your relationship like?"— Mark Mirra (verbatim, Plaud transcript)
"I would say ninety nine point nine percent of the time we ask some form of those four questions, pretty early in the process just to get a baseline understanding."— Mark Mirra (verbatim, Plaud transcript)

A structured intake framework Ali must complete before generating strategy. Maps to classification but is more specific than current implementation. Critical nuance from Tom: "There has to be some sort of foreplay before we get into those types of things too" — diagnostic questions must not be asked cold; conversational warm-up first.

Catalog status: NEW. Should be added as a required early-conversation coaching move. Overlaps with existing classification questions but adds "attractive alternatives" and "contract intent."

4. Conversational Information Extraction

"Ali should be tested on its ability to naturally extract information from a user, mimicking a real conversation."— Tom Hilgeman

Ali should not rely on users providing complete briefings. The coaching conversation itself should be the information-gathering mechanism. Changes both the product behavior and the eval methodology.

Catalog status: Coaching delivery pattern. Related to D5 (Three Stages of Facilitation) but more specific to information extraction.

5. Coach Debate as Training Data

"The best way for us to do this is we put two people that do this in a room together, and we talk back and forth. So like Mark and me or Scott and me or Mark and Scott."— Tom Hilgeman (verbatim, Plaud transcript)
"We're basically putting an idea on trial... I am the transactionalist, Mark is the kumbaya Harvard person, and then we basically debate it."— Tom Hilgeman (verbatim, Plaud transcript)

A novel knowledge capture method: structured disagreement between experts produces the nuanced coaching wisdom that transcripts from client sessions don't always surface. Tom confirmed coach-on-coach sessions yield more "distilled information" than full-day client workshops. Terrance endorsed, comparing to UChicago School of Economics thought experiments.

Catalog status: Knowledge acquisition methodology. Not a coaching move itself, but the source of future coaching moves.

Engineering Impact
What needs to change in the codebase based on this meeting

Priority 1: Transparency by Negotiation Type

Decision #1. Ali currently has no concept of modulating information-sharing advice by negotiation type. The "honesty" framing in the system prompt needs to become a "transparency" parameter driven by the Four Negotiation Types classification (Bargaining, Trading, Creating, Partnering).

Files affected: coaching-methodology.ts (position protection section), v3-braintrust-synced.ts (system prompt), classification logic.

Complexity: Medium. Requires mapping transparency levels to negotiation types and adjusting coaching guidance per type.

Priority 2: Four Required Diagnostic Questions

Decision #4. Mark defined four specific questions Ali must ask early: (1) terms being negotiated, (2) attractive alternatives, (3) contract intent (one-off vs LTA), (4) relationship status. These overlap with existing classification but are more specific.

Files affected: v3-braintrust-synced.ts (opening behavior), classification prompt, eval scenarios.

Complexity: Low-Medium. Can be added to opening instructions in system prompt. Need to integrate with existing classification flow.

Priority 3: Eval Methodology — Extraction Testing

Decision #3. Current eval scenarios provide full briefings upfront. New scenarios needed where the simulated user gives information piecemeal and Ali must ask for what it needs.

Files affected: eval-ali-multiturn.ts, eval-shared.ts (scenario definitions), new scenario briefings.

Complexity: Medium. User simulator needs to withhold info and respond only when asked. Different scenario structure.

Priority 4: Term Prioritization Question

Decision #2. Ali asks "which terms are you trying to maximize, and where can you be flexible?" early in the conversation. This directly prevents the backwards-pricing-tier problem.

Files affected: v3-braintrust-synced.ts (strategy generation instructions), coaching-methodology.ts.

Complexity: Low. Add to Ali's required early-conversation behavior in the system prompt.

Priority 5: Book Ingestion Pipeline

Decision #6 + #9. Ingest 3 negotiation books cover-to-cover into Ali's knowledge base. All ordered Feb 12:

  • Never Split the Difference (Chris Voss) — Arriving overnight. FBI tactical negotiation, emotional leverage, calibrated questions.
  • Getting to Yes (Fisher, Ury & Patton) — Arriving overnight. Harvard Negotiation Project, principled/integrative negotiation. The "kumbaya Harvard" pole Tom references.
  • 3-D Negotiation (Lax & Sebenius) — Arriving Feb 18-23. Technical frameworks, deal design. Tom: "3D negotiation is technical, and most negotiation books are a lot more communication psychological."

Pipeline (researched):

  1. Acquire digital text: Buy Kindle editions (~$25-45 total, all 3 available) → brew install --cask calibre + DeDRM plugin → import .azw3 → convert to .txt/.epub. Alt: Libby (library app) has free digital loans.
  2. Extract: pymupdf4llm (Python) → LLM-optimized markdown with chapter structure
  3. Chunk: 1,500–3,000 token semantic chunks per chapter, 10-20% overlap
  4. Extract coaching moves: Claude Haiku Batch API processes each chunk → structured JSON (coaching moves, frameworks, examples, quotes)
  5. Store: TypeScript files mirroring existing asf-sections.ts pattern → src/chatbot/knowledge/books/
  6. Review: HTML dashboard (existing docs/*.html pattern) — Mark/Tom approve/reject each extracted coaching move
  7. Inject: Static injection into system prompt alongside ASF sections (all 3 books ~320K tokens, fits in context window)

Cost: ~$25-45 Kindle editions + ~$2 Haiku extraction = ~$30-50 total. $0 ongoing.

Status: Physical books arriving (2 overnight, 1 Feb 18-23). Next step: buy Kindle editions, install Calibre, proof-of-concept with Voss Ch1.
